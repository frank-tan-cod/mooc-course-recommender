请帮我完整设计并实现一个课程设计项目，项目名称为：

《基于Spark的慕课课程协同过滤推荐系统设计与实现》

一、项目背景

本项目面向在线教育/慕课平台场景，利用 Apache Spark 大数据处理平台和 Spark MLlib 机器学习库，设计并开发一个简易的慕课课程个性化推荐系统。系统基于用户历史选课、学习行为或课程交互数据，构建用户-课程隐式偏好矩阵，并使用 ALS 协同过滤算法训练推荐模型，最终为用户生成 Top-N 个性化课程推荐结果。

系统需要满足课程设计任务要求，包括推荐数据采集与接入、数据清洗与预处理、智能化推荐模型训练、推荐结果存储、可视化展示、参数交互调整和应用案例演示。

二、技术栈要求

请使用以下技术栈：

1. Python 作为主要开发语言。
2. PySpark 作为大数据处理框架。
3. Spark MLlib 的 ALS 算法实现协同过滤推荐。
4. Streamlit 实现简易 Web 可视化交互界面。
5. Pandas 辅助处理小规模展示数据。
6. Plotly 或 Matplotlib 实现图表可视化。
7. Parquet/CSV 作为中间数据和推荐结果存储格式。
8. 如有必要，可使用 SQLite 存储推荐结果，但不是强制要求。

三、数据集要求

项目数据来源为慕课课程推荐场景数据，优先适配 MOOCCubeX 数据集，或者使用结构等价的模拟/抽样数据。

系统需要支持以下核心数据结构：

1. 用户数据：
   - user_id：用户ID
   - course_order：用户已选择或学习过的课程列表

2. 课程数据：
   - course_id：课程ID
   - course_name：课程名称
   - category：课程类别或领域
   - teacher：授课教师，可选
   - description：课程简介，可选

3. 用户-课程交互数据：
   - user_id
   - course_id
   - preference

如果原始数据没有显式评分，则需要将用户选课、观看视频、完成习题等行为转化为隐式偏好分数。例如：
- 选课行为 preference = 1.0
- 观看视频行为 preference += 0.5
- 完成习题行为 preference += 0.8
- 最终对 preference 做适当归一化或截断处理

四、系统功能要求

请实现以下功能模块：

1. 数据接入模块
   - 读取 data/raw/ 下的用户数据、课程数据和交互数据。
   - 支持 CSV 或 JSON 格式。
   - 如果没有真实数据，请提供一个可运行的示例数据生成脚本，生成模拟的用户、课程、交互数据。

2. 数据预处理模块
   - 去除缺失用户ID或课程ID的数据。
   - 去除重复的用户-课程交互记录。
   - 过滤交互过少的用户和课程。
   - 将字符串类型的 user_id 和 course_id 编码为 ALS 可用的整数索引。
   - 生成以下中间表：
     - interactions_clean
     - user_index_map
     - course_index_map
     - als_train_data

3. 数据统计分析模块
   - 统计用户数量、课程数量、交互记录数量。
   - 统计热门课程 Top 10。
   - 统计课程类别分布。
   - 统计用户选课数量分布。
   - 统计课程被学习次数分布。

4. ALS 协同过滤推荐模块
   - 使用 Spark MLlib ALS 训练模型。
   - 使用隐式反馈模式 implicitPrefs=True。
   - 支持以下参数配置：
     - rank
     - maxIter
     - regParam
     - topN
   - 训练完成后，为指定用户生成 Top-N 课程推荐。
   - 支持为所有用户批量生成推荐结果。
   - 推荐结果需要关联课程名称、课程类别、推荐得分等信息。

5. 模型评估模块
   - 如果可以划分训练集和测试集，请计算 RMSE 或其他合理指标。
   - 对隐式反馈推荐场景，可以同时展示推荐覆盖率、热门课程占比、推荐结果多样性等简单指标。
   - 保存不同参数组合下的模型运行结果，便于可视化对比。

6. 推荐结果存储模块
   - 将清洗后的交互数据保存到 data/processed/。
   - 将推荐结果保存到 data/output/。
   - 推荐结果字段包括：
     - user_id
     - course_id
     - course_name
     - category
     - recommendation_score
   - 保存格式优先使用 Parquet，同时可导出 CSV 方便查看。

7. Web 可视化交互模块
   使用 Streamlit 实现 app.py，（可视化框架与 Spark 集成要求：

Streamlit 页面不能只展示静态模拟图表，必须读取 Spark 数据处理流程输出的 processed/output 目录下的 Parquet 或 CSV 文件，包括清洗后的交互数据、课程统计结果、模型评估指标和推荐结果。模型训练页面点击训练按钮时，应调用 PySpark/Spark MLlib ALS 训练流程，并将新的推荐结果和指标写入 data/output/，然后由 Streamlit 页面重新读取并展示。Plotly 图表的数据来源必须来自 Spark 处理后的结果表，而不是手工写死的数据。）页面至少包含以下功能：

   A. 数据概览页面
   - 展示用户总数、课程总数、交互记录总数。
   - 展示热门课程 Top 10 柱状图。
   - 展示课程类别分布图。
   - 展示用户选课数量分布图。

   B. 模型训练页面
   - 提供 rank、maxIter、regParam、topN 参数输入控件。
   - 点击按钮后训练 ALS 模型。
   - 展示模型训练状态和评估指标。

   C. 个性化推荐页面
   - 用户可以输入或选择 user_id。
   - 系统展示该用户的 Top-N 推荐课程。
   - 展示推荐课程类别分布图。
   - 展示推荐分数柱状图。

   D. 结果分析页面
   - 展示不同参数设置下的模型指标对比。
   - 展示热门推荐和个性化推荐的对比。
   - 展示系统应用案例说明。
